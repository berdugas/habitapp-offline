2. Guided onboarding flow

The app helps the user create the first Focus habit by moving from identity, to daily action, to tiny action, to cue, to worst-day validation.

P0 Navigation gap

Today → Habit Detail is not obvious

The Habit Detail route is rich, but Today’s FocusCard is rendered as a View with Done/Skip buttons and no visible detail CTA. Add “View details” or make the card pressable.

P1 Product gap

Library is still placeholder

The Library tab describes future identity progress but does not yet show automatic, completed, or identity-strengthened habits.

P1 Copy mismatch

Archived habit reactivation is unclear

Settings says pause and resume habits, while Habit Detail says reactivation is coming in a future release. Align this behavior and copy.

Implemented strength

First-principles onboarding is strong

The flow uses identity, tiny action sizing, routine cueing, and worst-day validation before creating the first habit.

Implemented strength

Recovery loop is aligned with habit science

The app treats misses as recoverable, provides a make-it-smaller path, and avoids all-or-nothing streak punishment.

P2 Future opportunity

Weekly Review entry is buried

Weekly Review is available from Habit Detail, but not surfaced clearly in Today. Add a Today CTA when review is due.
